OAuth2 Authentication System

Authorization Code Flow com PKCE obrigatório, revogação em cascata, refresh token rotacionado e proteção por scopes. Construído com FastAPI e TDD.

FastAPI PKCE · RFC 7636 JWT · HS256 RFC 6749 RFC 7009 SQLAlchemy 2.0 SDD + TDD Python 3.12
Authorization Code + PKCE

Como funciona o fluxo

01 · Client
Gerar PKCE
code_verifier = random(43-128 chars) code_challenge = base64url(sha256(verifier))
02 · Client → Server
GET /oauth/authorize
?response_type=code &client_id=... &code_challenge=... &code_challenge_method=S256 &state=random
03 · Server
Autentica usuário e valida client
• verifica client_id e redirect_uri • checa escopo solicitado • armazena code_challenge
04 · Server → Client
302 Redirect
redirect_uri?code=abc123&state=xyz ↳ código de uso único, expira em 10 min
05 · Client → Server
POST /oauth/token
grant_type=authorization_code code=abc123 code_verifier=original ← prova PKCE client_id + client_secret
06 · Server valida PKCE
sha256(code_verifier) == code_challenge?
Se sim → emite access_token + refresh_token Se não → 400 invalid_grant
07 · Client
Acessa recurso protegido
GET /users/me Authorization: Bearer {access_token} ↳ scope "profile" obrigatório

Endpoints

Referência da API

POST/auth/register
Registra usuário. Email único, senha ≥ 8 chars. Retorna 201 com perfil.
M1
POST/auth/login
Autentica e retorna JWT de acesso. Credenciais inválidas → 401.
M2
GET/oauth/authorize
Inicia Authorization Code Flow com PKCE obrigatório (S256). Retorna 302.
M3
POST/oauth/token
Troca authorization_code por tokens (PKCE) ou renova via refresh_token.
M4 · M5
GET/users/me
Retorna perfil do usuário autenticado. Requer Bearer token com scope "profile".
M6
POST/oauth/revoke
Revoga access ou refresh token (RFC 7009). Cascata: revoga access tokens derivados.
M7
POST/demo/setup
Cria o cliente OAuth2 de demonstração com redirect_uri local.
Demo
GET/health
Health check. Retorna {"status":"ok"}.
